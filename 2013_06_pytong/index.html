Les administrateurs système font ça mieux que nous

On va parler de

La sécurisation de son ordinateur

Y'a pas que les virus dans la vie, y'a les trojan aussi

Le minimum

  • mot de passe (rappel: XKCD 936)
  • verouillage de la session lors des AFK
  • passphrase sur sa clé SSH
  • aucune info critique dans le code source
Non, 1234 n'est pas un mot de passe. h4x0r non plus.

Mais attention

  • pas de wifi public
  • pas d'envoi de données par mail
  • pas d'échange de mots de passe / info critique
  • pas de mot de passe partagé
  • procédures suivies par TOUT le monde
Le VPN c'est bien. C'est compliqué, mais c'est bien. Mais c'est compliqué.

La solution pragmatique

  • connexion aux serveurs en ssh par clé
  • pas de connexion aux serveurs
  • utilisation de scripts (astuce : django-runcommands)
  • utiliser son propre "paste" (0bin.net)
Tu crois que sécuriser ses données coûte cher ? Attends de t'en être fait voler pour voir

La solution pragmatique (DB)

  • pas de dump de la DB de production
  • ou alors anonymisé (user, profile)
  • utiliser un script de génération de données de test
  • utiliser http://factoryboy.rtfd.org
Tout anonymiser (nom, prénom, pass, adresse, mail, n° sécu ...) ou générer des données au hasard ?
